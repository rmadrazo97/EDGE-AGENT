# User

## Profile

- **Role:** Senior software engineer, solo operator
- **Trading experience:** Technical, data-driven, not a professional trader
- **Risk tolerance:** Conservative. Capital preservation over aggressive returns.

## Interfaces

- **Telegram:** Primary interface for quick status checks, pause/resume, kill commands
- **OpenClaw:** Deeper operations, debugging, strategy configuration
- **Terminal:** Direct access for development, deployment, log inspection

## Preferences

- Concise Telegram messages (scannable in 5 seconds)
- Numbers over narratives (show P&L, percentages, sizes)
- Confirmation required for any destructive action
- Prefers testnet validation before live changes
- Wants to understand why a trade was taken, not just that it was
