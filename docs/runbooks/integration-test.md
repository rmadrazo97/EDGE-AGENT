# Integration Test Runbook (Phase 2.4)

Run the full EDGE-AGENT system end-to-end on Binance testnet for 24-48 hours.

## Prerequisites

1. **Docker** running locally (`docker info` should succeed)
2. **Binance testnet credentials** configured in `.env`:
   - `BINANCE_TESTNET_API_KEY`
   - `BINANCE_TESTNET_API_SECRET`
   - `BINANCE_TESTNET_ENABLED=true`
3. **Telegram bot** configured in `.env`:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_OPERATOR_CHAT_ID`
4. **Moonshot API key** set in `.env`:
   - `MOONSHOT_API_KEY`
5. **Infrastructure up**: `make up` has been run and containers are healthy

## Pre-flight checks

```sh
# 1. Verify infrastructure is healthy
make smoke

# 2. Verify testnet balances (run a single analyst cycle to confirm API connectivity)
make analyst-once

# 3. Verify market data is flowing (check output for price data)
make trader-once
```

All three commands should complete without errors.

## Starting the integration test

Use the runner script to start all three agents concurrently:

```sh
make integration-test
```

This runs `infra/scripts/integration-test.sh`, which:
- Verifies prerequisites (Docker, smoke checks, env vars)
- Starts the analyst agent in continuous mode (background)
- Starts the trader agent in continuous mode (background)
- Starts the reporter/Telegram bot (background)
- Writes PIDs to `runtime/integration-test-pids.txt`
- Traps SIGINT/SIGTERM for clean shutdown

## What to monitor via Telegram

During the 24-48h run, use Telegram as the sole monitoring channel:

- **Trade notifications**: opens, closes, stop-loss triggers
- **Approval requests**: out-of-bounds proposals requiring operator approval
- **Daily reports**: position summary, P&L, risk status
- **OpenClaw commands**: test natural language queries ("what are my open positions?", "pause trading", "resume trading")

## Checking logs

If you need to inspect logs directly:

```sh
# Analyst logs (one JSONL per cycle)
ls runtime/analyst/*.jsonl

# Trader logs (one JSONL per cycle)
ls runtime/trader/*.jsonl

# Policy audit log
cat runtime/audit/policy.jsonl

# Telegram audit log
cat runtime/audit/telegram.jsonl
```

Each JSONL file contains timestamped events. Key event types:

| Log directory | Event types |
|---|---|
| `runtime/analyst/` | `analysis_started`, `signal_generated`, `signal_filtered` |
| `runtime/trader/` | `trade_opened`, `position_closed`, `position_held`, `trade_rejected`, `approval_resolved` |
| `runtime/audit/` | Policy decisions, Telegram message records |

## Success criteria checklist

From the Phase 2.4 PRD -- all must pass:

- [ ] System runs 24h without crashing
- [ ] Analyst produces signals at expected interval
- [ ] Trader executes trades within risk bounds
- [ ] Stop losses are placed on every position
- [ ] Daily loss limit is respected
- [ ] Push notifications arrive for all trade events
- [ ] Approval flow works for out-of-bounds proposals
- [ ] Daily report is accurate (matches actual positions/P&L)
- [ ] OpenClaw responds to natural language queries via Telegram
- [ ] Pause/resume via OpenClaw toggles policy correctly
- [ ] System recovers from agent restart without losing position state
- [ ] No unhandled exceptions in logs over 24h

## How to stop cleanly

Press `Ctrl+C` in the terminal running `make integration-test`. The script traps the signal and sends SIGTERM to all child processes.

Alternatively, if the terminal is lost:

```sh
# Read saved PIDs and kill them
cat runtime/integration-test-pids.txt | xargs kill
```

## Post-run analysis

Generate a summary report:

```sh
make integration-report
```

This runs `infra/scripts/integration-test-report.sh`, which outputs:

- Total signals generated by the analyst
- Total trades opened and closed by the trader
- Win rate (closed trades with positive P&L / total closed)
- Total realized P&L
- Error count across all logs
- Any unhandled exceptions found

### Manual inspection

```sh
# Count signals
grep -c '"signal_generated"' runtime/analyst/*.jsonl | tail -1

# Count trades opened
grep -c '"trade_opened"' runtime/trader/*.jsonl | tail -1

# Count trades closed
grep -c '"position_closed"' runtime/trader/*.jsonl | tail -1

# Look for errors
grep -i '"error"' runtime/analyst/*.jsonl runtime/trader/*.jsonl | grep -v 'null'
```

### Bug tracking

Document all issues found during the integration run:
- **Critical** (fix before production): positions not tracked, risk rules bypassed, notifications missing
- **Non-critical** (defer): formatting issues, timing drift, cosmetic problems
