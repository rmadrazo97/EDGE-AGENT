# Incident Log

Record of issues encountered and their resolutions. See `runbooks/incident-response.md` for the response process.

## Template

```
### INC-XXXX — YYYY-MM-DD HH:MM UTC
- **Severity:** low / medium / high / critical
- **Summary:** one-line description
- **Impact:** what was affected (positions, P&L, availability)
- **Root cause:** what went wrong
- **Resolution:** what was done to fix it
- **Prevention:** what changed to prevent recurrence
```
